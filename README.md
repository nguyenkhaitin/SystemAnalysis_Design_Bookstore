> *A comprehensive Software Requirement Specification (SRS) and logical blueprint for a modern retail bookstore enterprise.*

This project was developed for the **System Analysis and Design** course at the **University of Social Sciences and Humanities (USSH)**. It demonstrates a professional-grade analytical lifecycle, from initial requirement elicitation to complex logical modeling and physical database implementation. The project focuses on bridging the gap between business needs and technical solutions, earning a **9.0/10 grade**.

---

### My Role: Business Analyst & Database Architect
In this project, I acted as the primary bridge between the "client" (the instructor) and the technical design, focusing on the core BA competencies:

* **Requirement Elicitation:** Interfaced directly with stakeholders to gather business requirements, define system scope, and identify pain points in bookstore operations.
* **UML & Logical Modeling:** Translated complex business rules into structured technical models. I designed the **BFD (Business Functional Decomposition)**, **Use Case Diagrams**, and **Activity Diagrams** to map out every possible user interaction.
* **Functional Specification (SRS):** Authored detailed Use Case Specifications and designed static UI mockups using **HTML/CSS** to visualize the proposed solution for stakeholders.
* **Process & Data Architecture:** Built multi-layered **Data Flow Diagrams (DFD)** to trace data movement across the system and utilized **Decision Trees** to define complex operational logic (e.g., inventory restocking rules, discount policies).
* **Database Engineering:** Designed and implemented a high-integrity relational database in **MSSQL Server**, involving table creation, relationship mapping (ERD/RDM), and the enforcement of strict data integrity constraints.

---

### Key Objectives
* **Process Optimization:** Streamline core bookstore activities, including procurement, sales, inventory management, and reporting.
* **Rigorous Documentation:** Produce a high-quality SRS that serves as a single source of truth for development.
* **Logical Accuracy:** Ensure all business rules (Decision Trees) are accurately reflected in the database and process flows.

---

### Analytical Framework
The project is structured into three analytical pillars:

#### 1. Behavioral & Functional Modeling
* Defined actors and interactions through professional UML standards.
* Mapped complex workflows (e.g., Book Return, Staff Shift Management) via Activity Diagrams.

#### 2. Data Flow & Structure
* **DFD (Level 0, 1, 2):** Describing how data is processed, stored, and retrieved.
* **ERD/RDM:** A normalized (3NF) relational schema ensuring zero data redundancy.

#### 3. Visual Prototype (UI Mockup)
* Developed a static front-end interface using HTML/CSS to demonstrate the user experience (UX) and system flow to stakeholders.
* *Note: The current web interface is a high-fidelity static prototype and is not yet connected to the SQL backend.*
